## Prioritization Principle

Fix production safety, contamination, source traceability, and citation correctness before polishing document style. A shorter, prettier DOCX is still unsafe if it cites weak evidence, leaks stale assets, or proceeds from a blocked handoff.

## Recommended First Implementation Batch

Start with Batch 1.

Why:

- The diagnostic run was structurally successful but explicitly `production_valid: false`.
- Step 3 was `BLOCK`, the consistency matrix was `blocked`, and traceability score was only 53.33.
- User-facing quality issues such as duplicate TOC, title style, redundancy, and visuals matter, but they should be fixed after the pipeline can prove that every generated claim comes from the current run and from usable sources.
- Contamination control must come early because old intake/assets/equations leaking into a new run invalidates every later QA signal.

Recommended Batch 1 slice:

1. Add production eligibility gates for degraded/blocked handoffs and blocked Lab B matrix.
2. Add fresh-run isolation checks so no old intake, assets, equations, or latest-path artifacts can leak into the output.
3. Fix citation and reference-location semantics before changing prose style.
4. Separate public academic appendices from internal/developer traceability.

## Definition of Done for This Backlog Phase

- Diagnostic mode remains available, but production mode cannot produce DOCX from degraded or contaminated inputs.
