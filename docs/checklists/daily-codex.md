# Daily Codex Checklist

Before starting:

- confirm the task belongs to one subsystem
- choose the right thread family
- decide whether a new temporary thread is needed
- decide whether the task needs a dedicated worktree

While working:

- keep logs out of chat unless they are short and diagnostic
- store raw evidence in `artifacts-local/`
- avoid mixing design, debugging, and implementation
- update only one dominant area per thread

Before closing:

- verify the change
- write a thread brief if the result is durable
- list follow-ups explicitly
- close or retire the temporary thread
